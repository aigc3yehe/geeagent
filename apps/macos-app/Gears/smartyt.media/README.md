# SmartYT Media Gear

SmartYT Media is a native Gee Gear adapted from the reference SmartYT project.
It keeps the SmartYT core workflow while fitting Gee's Gear platform:

- accept a URL
- sniff media metadata with `yt-dlp`
- download audio or video
- extract transcript text from platform subtitles when available
- optionally fall back to a local Whisper command when the user has installed one

## Runtime Dependencies

The Gear declares these dependencies in `gear.json`:

- `yt-dlp`
- `ffmpeg`
- `ffprobe`

Gee checks the dependencies when the Gear is opened from the Gears catalog. If
Homebrew is available, Gee can install missing `yt-dlp` and `ffmpeg`
dependencies with the registered recipes.

## Data Locations

Job state is written outside the Gear package:

```text
~/Library/Application Support/GeeAgent/gear-data/smartyt.media/jobs/
```

Downloaded media, extracted subtitles, and transcript text default to:

```text
~/Downloads/SmartYT/<job-id>/
```

Agent calls may pass an explicit `output_dir` for download or transcription
jobs. If no output directory is specified, Gee uses `~/Downloads/SmartYT`. The
Gear package itself remains copy-installable and should not store user
downloads.

## Agent Capabilities

The Gear exposes three progressively disclosed capabilities:

- `smartyt.sniff`
- `smartyt.download`
- `smartyt.transcribe`

The Gear returns structured execution results only. Final user-facing prose must
be generated by the active agent/LLM after the host bridge returns the
structured result.
