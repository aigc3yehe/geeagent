# Twitter Capture Gear

Twitter Capture is a native Gee Gear adapted from the reference
`workbench-app` Twitter capture flow. It uses a small Twikit sidecar and saves
results into a local file database owned by the Gear.

## Features

- Fetch one tweet from a tweet URL.
- Fetch the first N tweets from a Twitter/X List URL.
- Fetch the first N tweets from a username or profile URL.
- Save task metadata and result tweets in the Gear data directory.
- Expose the same three operations to the Gee agent through Gear capabilities.

## Cookie Requirement

Twitter/X content fetching requires a valid logged-in cookie JSON file. The
cookie file may be either:

- a JSON object such as `{ "ct0": "...", "auth_token": "...", "twid": "..." }`
- a browser-exported cookie array with `{ "name": "...", "value": "..." }`

The native Gear UI lets the user select a default cookie file. Agent calls may
also pass `cookie_file` explicitly.

## Data Location

Task records and captured tweet results are stored in:

```text
~/Library/Application Support/GeeAgent/gear-data/twitter.capture/tasks/<task-id>/task.json
```

This is intentionally inside the Gear app's internal task result database, not
the package folder. The Gear package remains copy-installable.

## Agent Capabilities

- `twitter.fetch_tweet`
- `twitter.fetch_list`
- `twitter.fetch_user`

Each capability creates and runs a Gear task, persists the result, and returns
structured result data. Final user-facing prose must be generated by the active
agent/LLM after the structured Gear result is returned.
